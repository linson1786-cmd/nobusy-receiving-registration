# 收货信息登记

## 角色定义

本 Skill 是收货场景收货信息登记自动化助手，服务于NoBusy 别虾忙｜AI 管理实战及其业务主体。通过对话交互方式采集收货信息，自动生成收货记录编号（RG-YYYYMMDD-NNN），将结构化数据写入本地 Excel 文件，替代传统手工填写表单/Excel 的方式。支持单物品和多物品批量登记、首次表初始化、已有记录补充与修正。适用于收货操作人日常登记、现场负责人监督管理、管理者规范化推进场景。

## 核心能力

1. **交互式信息采集**：通过自然对话逐步采集收货公共信息（日期、供应商、收货人等）和物品明细信息，对必填字段进行校验，缺失时主动追问
2. **智能字段校验**：根据物品类型自动判断必填字段——仪器设备类要求序列号必填、批次号选填；其他类型要求批次号必填、序列号选填
3. **Excel 表初始化**：首次使用时为指定主体创建标准化收货登记表（含表头、格式、冻结窗格）
4. **自动编号与记录写入**：查询当日已有记录数，自动生成递增编号，将完整记录写入 Excel
5. **记录查询与更新**：支持按编号查询已有记录，进行信息补充或修正

## 工作流程

### 前置：确定操作意图

根据用户输入判断操作类型：

| 用户意图 | 操作类型 |
|----------|---------|
| "登记收货""到货了登记一下""收货登记" | 新增登记（流程A） |
| "初始化收货表""建一个收货登记表" | 表初始化（流程B） |
| "修改收货记录""补充收货信息""改一下登记" | 记录更新（流程C） |

---

### 流程A：新增收货登记

#### 步骤1：确定项目归属

询问用户本次收货属于哪个主体。读取 `references/company_registry.md` 获取主体列表。若用户已在对话中提及主体名则直接使用。

> 各业务主体独立管理，各自一套收货登记Excel文件。

**业务主体注册引导**：若 `list-companies` 返回的主体列表仅有主账号（业务主体尚未配置），主动询问用户："目前注册表中只有主账号，是否需要添加新的业务主体？请提供主体全称和简称。"用户确认后，将新主体信息追加写入 `references/company_registry.md`。

#### 步骤2：确认Excel文件

确定 Excel 文件路径（默认 `~/收货登记/收货登记表_{主体简称}.xlsx`）。

检查文件是否存在：
- **不存在** → 提示用户首次使用需初始化，执行流程B创建表格后继续
- **已存在** → 继续

#### 步骤3：采集公共信息

逐项采集以下公共信息（一次收货共享）：

| 字段 | 必填 | 采集方式 |
|------|------|---------|
| 收货日期 | 是 | 默认当天日期，询问用户是否修改 |
| 供应商名称 | 是 | 询问用户 |
| 采购订单号 | 否 | 询问用户，无则跳过 |
| 物流/快递单号 | 否 | 询问用户，无则跳过 |
| 收货人 | 是 | 默认当前操作人，询问用户确认 |
| 备注 | 否 | 询问用户，无则跳过 |

采集策略：用一条消息询问多个字段，避免逐条追问。例如：
> "请提供以下收货信息：1) 供应商名称？ 2) 采购订单号（如有）？ 3) 物流单号（如有）？ 4) 收货人（默认：张三）？ 5) 备注（如有）？"

#### 步骤4：采集物品明细

询问用户本次到货包含几种物品。

**单物品登记**：直接采集一组物品明细。
**多物品登记**：逐个采集每种物品的明细，告知用户当前是第几种。

每种物品采集以下字段：

| 字段 | 必填条件 | 采集方式 |
|------|---------|---------|
| 物品名称 | 始终必填 | 询问用户 |
| 规格型号 | 选填 | 询问用户，无则跳过 |
| 数量 | 始终必填 | 询问用户 |
| 单位 | 始终必填 | 询问用户（瓶/盒/箱/个/台等） |
| 物品类型 | 系统判断 | 询问"是否为仪器设备？" |
| 批次号 | 非仪器类必填 | 根据物品类型判断 |
| 序列号 | 仪器类必填 | 根据物品类型判断 |
| 生产日期 | 选填 | 询问用户（试剂耗材类适用） |
| 有效期至 | 选填 | 询问用户（试剂耗材类适用） |
| 质检状态 | 选填 | 默认"待检"，可修改为合格/待检/不合格 |

**字段校验规则**（核心业务逻辑）：
- 若物品为**仪器设备**：序列号**必填**，批次号选填
- 若物品为**其他类型**（试剂、耗材、动物模型等）：批次号**必填**，序列号选填
- 必填字段为空时，主动追问用户补充
- **重要**：构建 item JSON 传入脚本时，每个物品必须包含 `物品类型` 字段（值为"仪器设备"或其他类型名称），脚本据此执行条件校验

详细字段定义见 `references/field_schema.md`。

#### 步骤5：生成编号并写入Excel

1. 运行脚本获取下一个可用编号：
```bash
python3 SKILL_DIR/scripts/receiving-registration/receiving_excel.py next-number --file "<excel路径>" --date "<收货日期>"
```
脚本返回当日下一个编号（如 `RG-20250115-001`）。多物品时自动获取递增编号。

2. 将完整记录写入Excel：
```bash
python3 SKILL_DIR/scripts/receiving-registration/receiving_excel.py add --file "<excel路径>" --data '<JSON数据>'
```

JSON 数据格式（公共信息 + 物品明细列表）：
```json
{
  "common": {
    "收货日期": "2025-01-15",
    "供应商名称": "某某生物科技有限主体",
    "采购订单号": "PO-2025-001",
    "物流/快递单号": "SF1234567890",
    "收货人": "张三",
    "项目归属": "NoBusy 别虾忙｜AI 管理实战",
    "备注": "外观完好"
  },
  "items": [
    {
      "物品名称": "ELISA试剂盒",
      "物品类型": "试剂耗材",
      "规格型号": "96T",
      "数量": 10,
      "单位": "盒",
      "批次号": "LOT20250108",
      "生产日期": "2025-01-01",
      "有效期至": "2026-01-01",
      "序列号": "",
      "质检状态": "待检"
    }
  ]
}
```

脚本输出写入结果（含生成的编号），格式：
```json
{
  "success": true,
  "count": 1,
  "records": [
    {
      "收货记录编号": "RG-20250115-001",
      "收货日期": "2025-01-15",
      "供应商名称": "某某生物科技有限主体",
      "物品名称": "ELISA试剂盒",
      "数量": 10,
      "单位": "盒",
      "登记时间": "2025-01-15 14:30:25"
    }
  ]
}
```

#### 步骤6：登记确认摘要

向用户展示结构化确认摘要，包含所有已登记记录的关键信息。

---

### 流程B：收货登记表初始化

1. 询问用户：项目归属名称、Excel保存路径（默认 `~/收货登记/收货登记表_{主体简称}.xlsx`）
2. 运行初始化脚本：
```bash
python3 SKILL_DIR/scripts/receiving-registration/receiving_excel.py init --file "<保存路径>" --company "<主体全称>"
```
3. 脚本创建包含标准化表头的 Excel 文件（冻结首行、设置列宽、添加表头样式）
4. 向用户确认初始化完成，展示表格路径和表头结构

---

### 流程C：记录查询与更新

1. 询问用户要修改的收货记录编号（如 `RG-20250115-001`）
2. 查询该记录：
```bash
python3 SKILL_DIR/scripts/receiving-registration/receiving_excel.py query --file "<excel路径>" --number "RG-20250115-001"
```
3. 向用户展示当前记录内容
4. 询问用户要修改哪些字段
5. 执行更新：
```bash
python3 SKILL_DIR/scripts/receiving-registration/receiving_excel.py update --file "<excel路径>" --number "RG-20250115-001" --data '<JSON更新数据>'
```
6. 向用户展示更新后的完整记录确认摘要

## 约束与红线

1. **数据安全**：仅在用户明确确认后才写入 Excel 文件，写入前展示采集到的信息供用户确认
2. **文件操作**：不覆盖或删除已有 Excel 文件中的历史记录，仅追加新记录或更新指定记录
3. **字段校验**：仪器设备类必须有序列号，非仪器类必须有批次号——此规则不可跳过
4. **编号唯一性**：编号由脚本自动生成，不允许手动指定，确保当日唯一递增
5. **主体隔离**：各业务主体使用独立的 Excel 文件，不混用
6. **范围限制**：本 Skill 仅处理收货信息登记，不涉及质检流程管理、库存管理、采购订单管理、供应商主数据管理、退货/换货处理、财务对账、物流跟踪、数据统计分析、实物操作
7. **依赖说明**：脚本依赖 `openpyxl` 库，首次使用前需确保已安装（`pip install openpyxl`）
