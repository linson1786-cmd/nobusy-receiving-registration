## 项目简介

本项目为 WorkBuddy 的 Skill 插件，用于仓储部门收货信息的交互式采集、结构化记录和自动化登记。通过对话采集收货信息，自动生成编号，写入本地 Excel 文件，替代传统手工填写表单/Excel 的方式。支持单物品/多物品批量登记、首次表初始化、已有记录补充修正。

## 快速开始（3 步上手）

### 第 1 步：安装依赖

```bash
pip install openpyxl
```

### 第 2 步：部署 Skill

```bash
cd scripts
python3 deploy.py
```

部署工具会自动将 Skill 文件同步到 `~/.workbuddy/skills/receiving-registration/`。

### 第 3 步：开始使用

**初始化收货表**：在 WorkBuddy 对话中说"初始化收货表"，按引导提供主体名称和保存路径。

**收货登记**：在 WorkBuddy 对话中说"收货登记"或"到货了登记一下"即可触发登记流程。

**修改记录**：说"修改收货记录"或"补充收货信息"，提供记录编号进行更新。

## 核心业务规则

### 物品类型与必填字段

| 物品类型 | 批次号 | 序列号 | 典型物品 |
|---------|--------|--------|---------|
| 仪器设备 | 选填 | **必填** | 离心机、移液器、显微镜等 |
| 其他类型 | **必填** | 选填 | 试剂、耗材、动物模型等 |

### 编号规则

```
RG-YYYYMMDD-NNN
```
- `RG`：收货固定前缀
- `YYYYMMDD`：收货日期
- `NNN`：当日序号，3位零填充，从001递增

### 主体隔离

各业务主体使用独立的 Excel 文件，命名格式：`收货登记表_{主体简称}.xlsx`，默认存储路径：`~/收货登记/`。

## 版本管理说明

### 版本号格式

- **V1.0** = 初始发布（当前版本）
- **V1.1** = 小功能更新或 bug 修复
- **V2.0** = 重大功能更新

### 自动更新机制

运行 `python3 scripts/deploy.py` 时：

1. 读取当前版本（`scripts/VERSION`）
2. 对比已部署版本（`~/.workbuddy/skills/.../scripts/VERSION`）
3. 若有新版本，自动备份旧文件到 `.backup/<version>/`
4. 更新 Skill 文件
5. **绝不触碰用户数据**（`~/收货登记/*.xlsx`）

### 版本回退

```bash
cd ~/.workbuddy/skills/receiving-registration/.backup/V1.0_YYYYMMDD_HHMMSS
cp *.py ../../scripts/
cp ../SKILL.md ../../
```

## 常见问题

**Q: 多人可以同时使用吗？**  
A: 可以。每位使用人各自的 Excel 文件独立，Skill 代码共享。版本更新只更新脚本代码，不影响各使用人的收货登记数据。

**Q: 如何添加新的业务主体？**  
A: 首次使用时，系统会引导补充主体信息。也可以直接编辑 `references/company_registry.md` 添加。

**Q: Excel 文件存哪里？**  
A: 默认 `~/收货登记/收货登记表_{主体简称}.xlsx`，初始化时可自定义路径。
