# 收货登记字段定义

## Excel 表头结构（18列）

| 列序 | 字段名 | 英文键名 | 数据类型 | 必填 | 说明 |
|------|--------|---------|---------|------|------|
| A | 收货记录编号 | record_number | 字符串 | 系统生成 | 格式：RG-YYYYMMDD-NNN |
| B | 收货日期 | receive_date | 日期 | 是 | YYYY-MM-DD 格式，默认当天 |
| C | 供应商名称 | supplier_name | 字符串 | 是 | |
| D | 采购订单号 | po_number | 字符串 | 否 | |
| E | 物流/快递单号 | tracking_number | 字符串 | 否 | |
| F | 收货人 | receiver | 字符串 | 是 | 默认当前操作人 |
| G | 物品名称 | item_name | 字符串 | 是 | |
| H | 规格型号 | specification | 字符串 | 否 | |
| I | 数量 | quantity | 数值 | 是 | 正整数或正小数 |
| J | 单位 | unit | 字符串 | 是 | 瓶/盒/箱/个/台等 |
| K | 批次号 | batch_number | 字符串 | 条件必填 | 见下方校验规则 |
| L | 生产日期 | production_date | 日期 | 否 | 试剂耗材类适用 |
| M | 有效期至 | expiry_date | 日期 | 否 | 试剂耗材类适用 |
| N | 序列号 | serial_number | 字符串 | 条件必填 | 见下方校验规则 |
| O | 质检状态 | qc_status | 字符串 | 否 | 合格/待检/不合格，默认"待检" |
| P | 项目归属 | company | 字符串 | 系统生成 | 用于区分业务主体 |
| Q | 登记时间 | register_time | 日期时间 | 系统生成 | YYYY-MM-DD HH:MM:SS |
| R | 备注 | remarks | 字符串 | 否 | |

## 字段分类说明

### 公共信息（一次收货共享）

以下字段在同一次收货中所有物品共享：
- 收货日期
- 供应商名称
- 采购订单号
- 物流/快递单号
- 收货人
- 项目归属
- 备注

### 物品明细信息（每个物品一组）

以下字段每个物品独立填写：
- 物品名称
- 规格型号
- 数量
- 单位
- 批次号
- 生产日期
- 有效期至
- 序列号
- 质检状态

### 系统自动生成字段

- 收货记录编号：脚本自动生成
- 登记时间：脚本自动填充当前时间
- 项目归属：根据用户选择的主体自动填充

## 核心校验规则

### 物品类型与条件必填字段

| 物品类型 | 批次号 | 序列号 | 典型物品 |
|---------|--------|--------|---------|
| 仪器设备 | 选填 | **必填** | 离心机、移液器、显微镜、培养箱、分析天平等 |
| 试剂耗材 | **必填** | 选填 | ELISA试剂盒、抗体、培养基、离心管、吸头等 |
| 动物模型 | **必填** | 选填 | 实验小鼠、大鼠等 |
| 其他 | **必填** | 选填 | 无法归类的物品 |

### 判断流程

```
用户输入物品名称后：
1. 询问用户："该物品是否为仪器设备？"
2. 若"是"（仪器设备）：
   → 序列号必填（追问直至提供）
   → 批次号选填（可跳过）
3. 若"否"（非仪器设备）：
   → 批次号必填（追问直至提供）
   → 序列号选填（可跳过）
```

### 数量校验

- 数量必须为正数（正整数或正小数）
- 常见单位参考：瓶、盒、箱、个、台、支、包、袋、升(L)、毫升(mL)、克(g)、千克(kg)

### 日期格式

- 收货日期：`YYYY-MM-DD`（如 2025-01-15）
- 生产日期：`YYYY-MM-DD`
- 有效期至：`YYYY-MM-DD`
- 登记时间：`YYYY-MM-DD HH:MM:SS`（如 2025-01-15 14:30:25）

### 质检状态枚举值

| 值 | 说明 |
|----|------|
| 待检 | 默认值，尚未进行质检 |
| 合格 | 已质检且通过 |
| 不合格 | 已质检但未通过 |

## 收货记录编号规则

### 格式

```
RG-YYYYMMDD-NNN
```

- `RG`：收货（Receiving）固定前缀
- `YYYYMMDD`：收货日期，8位
- `NNN`：当日序号，3位零填充，从001开始递增

### 示例

| 编号 | 含义 |
|------|------|
| RG-20250115-001 | 2025年1月15日第1条收货记录 |
| RG-20250115-002 | 2025年1月15日第2条收货记录 |
| RG-20250116-001 | 2025年1月16日第1条收货记录 |

### 生成逻辑

1. 读取目标 Excel 文件中所有记录
2. 筛选出收货日期与目标日期相同的记录
3. 计算已有记录数 N
4. 下一个编号为 `RG-{YYYYMMDD}-{(N+1):03d}`
5. 多物品登记时，从该编号开始依次递增
