## 各目录说明

### 1. 源文件/

**用途**：存放开发中的原始代码，是开发工作的主目录。

**规则**：
- 每个版本一个子目录（`V1.0/`, `V1.1/`, `V2.0/`）
- 包含 `scripts/`（脚本代码）和 `references/`（参考文档）
- 只包含源代码，不包含缓存（`__pycache__`）
- 修改代码时在此目录修改，测试通过后再打包到交付包

### 2. 交付包/

**用途**：存放对外发布的完整 Skill 包，使用人直接复制此目录即可使用。

**规则**：
- 每个版本一个子目录，包含完整文件
- 包含：`README.md` + `SKILL.md` + `交付说明.md` + `scripts/` + `references/`
- 不包含开发相关文件（测试记录、项目档案等）

### 3. 版本归档/

**用途**：存放历史版本的压缩包，用于长期存档和回溯。

**规则**：
- 每次发布后在交付包目录创建压缩包
- 命名格式：`V{版本号}-{YYYYMMDD}.zip`
- 永久保留，不可删除

### 4. 项目管理/

**用途**：存放所有项目管理相关的文档和记录。

**子目录**：

#### 4.1 版本发布记录/
记录每次版本发布的详细信息：发布日期、版本号、负责人、新增功能、修复 bug、已知问题、使用说明变更。

#### 4.2 需求与计划/
记录需求收集和版本规划：用户需求收集表、版本规划表、功能优先级排序、开发计划时间表。

#### 4.3 测试记录/
记录测试过程和结果：测试用例设计、测试结果记录、Bug 修复记录、回归测试记录。

#### 4.4 使用反馈/
记录用户使用过程中的反馈：用户反馈收集表、问题跟踪表、改进建议记录、常见问题 FAQ。

## 版本管理流程

### 开发流程

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│   需求收集   │ → │   代码开发   │ → │   测试验证   │ → │   发布交付   │
│  (需求与计划) │    │  (源文件/Vx) │    │  (测试记录)  │    │  (交付包/Vx) │
└─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
                                                          ↓
                                                   ┌─────────────┐
                                                   │   版本归档   │
                                                   │  (版本归档)   │
                                                   └─────────────┘
```

### 发布流程

1. **开发完成** → 在 `源文件/Vx.x/` 中完成代码开发
2. **测试通过** → 填写 `项目管理/测试记录/Vx.x-测试记录.md`
3. **创建交付包** → 复制到 `交付包/Vx.x/`
4. **编写交付说明** → 创建 `Vx.x-交付说明.md`
5. **归档** → 压缩 `交付包/Vx.x/` 到 `版本归档/Vx.x-YYYYMMDD.zip`
6. **记录发布** → 填写 `项目管理/版本发布记录/Vx.x-发布记录.md`
7. **通知用户** → 告知用户有新版本可用，提供更新指引
8. **部署** → 运行 `python3 scripts/deploy.py` 同步到 `~/.workbuddy/skills/`

## 命名规范

### 文件命名

| 类型 | 命名格式 | 示例 |
|------|----------|------|
| 版本目录 | `V{主版本}.{次版本}/` | `V1.0/`, `V1.1/`, `V2.0/` |
| 交付说明 | `V{版本}-交付说明.md` | `V1.0-交付说明.md` |
| 发布记录 | `V{版本}-发布记录.md` | `V1.0-发布记录.md` |
| 测试记录 | `V{版本}-测试记录.md` | `V1.0-测试记录.md` |
| 版本归档 | `V{版本}-{YYYYMMDD}.zip` | `V1.0-20260623.zip` |
| 需求文档 | `YYYY-MM-DD-需求收集.md` | `2026-06-23-需求收集.md` |
| 反馈记录 | `YYYY-MM-DD-反馈记录.md` | `2026-06-23-反馈记录.md` |

### 版本号规则

- **V1.0**：初始发布
- **V1.1**：小功能更新或 bug 修复（保持向后兼容）
- **V2.0**：重大功能更新（可能不兼容旧版本）
- 紧急修复：V1.0.1（仅在 V1.0 基础上 hotfix）

## 使用指南

### 开发新功能

1. 在 `源文件/V1.0/scripts/` 中修改代码
2. 测试通过后，复制到 `交付包/V1.0/scripts/`
3. 更新 `V1.0-交付说明.md` 中的版本说明
4. 归档并发布

### 发布新版本

1. 创建 `源文件/V1.1/` 目录
2. 复制 V1.0 代码到 V1.1，在此基础修改
3. 开发完成后，创建 `交付包/V1.1/`
4. 创建 `V1.1-交付说明.md`
5. 压缩归档到 `版本归档/V1.1-YYYYMMDD.zip`
6. 填写 `项目管理/版本发布记录/V1.1-发布记录.md`
7. 运行 `python3 scripts/deploy.py` 部署
